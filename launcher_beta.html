LauncherBeta

A simple, modern Android launcher built with Jetpack Compose.

Project Overview

Experience the Cleanest Android UI

LauncherBeta is an experimental, modern launcher built from the ground up using Jetpack Compose and Material 3. It focuses on providing a fluid, customizable experience while showcasing the latest Android development practices.

🚀 Core Features

  • Intelligent App Drawer: Quickly browse and search through all your installed applications with a fluid, state-aware UI.
  • Home Screen Customization: Add app shortcuts and folders for a personalized organization strategy.
  • Full Widget Support: Seamlessly integrate your favorite tools directly onto your home screen using the system widget picker.
  • Modern Navigation: Built with the latest androidx.navigation3 library for robust, predictable screen transitions.
  • Adaptive Layouts: Designed to scale beautifully from compact phones to large foldable devices.
  • Reliable Persistence: Uses Jetpack DataStore for efficient, type-safe layout saving and loading.

Technical Stack

Modern Android Architecture

LauncherBeta is built as a reference project for high-performance UI and clean architecture in 2026.

  • Jetpack Compose: 100% declarative UI for unmatched development speed and reliability.
  • Material 3 Design: Leveraging the latest Design System for adaptive colors and modern typography.
  • Navigation3: Implementing the latest generation of Android navigation for state management.
  • Jetpack DataStore: Modern, asynchronous data storage for high-performance preference management.
  • Kotlinx Serialization: Type-safe JSON serialization for complex layout configurations.
Project Structure

Engineered for Scalability

The codebase follows a modular structure that prioritizes separation of concerns and testability.

📂 Layer Overview

  • data/: Repositories (App, Home, Widget) and local data models.
  • ui/: Compose-based screens and components.
    • home/: Main workspace and widget hosting logic.
    • drawer/: App listing and search implementation.
    • theme/: Dynamic Material 3 configuration.
  • navigation/: Centralized destination definitions using Navigation3.
