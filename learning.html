1. 🌟 Project Overview

EchoCustom Pro is a modular Android personalization suite and launcher. It combines several high-level features:

  • Sound Mapping: Custom audio overrides for notifications on a per-app/per-contact basis.
  • Smart Launcher: A feature-rich home screen with a "Maid" assistant, modular dashboard, and resource monitoring.
  • Elder Mode: A specialized, high-visibility UI for simplified accessibility.
  • Privacy & Security: Call screening and accessibility-based gestures.
Note: The Niagara Launcher/ directory in the root is a separate reference project (uIndex Launcher) and is not part of the Gradle build.

2. 🏗 Architecture & Tech Stack

The project follows modern Android development practices:

  • Language: 100% Kotlin (with some Java for legacy widget support).
  • Architecture: MVVM (Model-ViewModel-Intent) with a reactive data flow.
  • UI Framework: Jetpack Compose (Material 3) for all modern screens; XML for legacy widgets and TutorialActivity.
  • Dependency Injection: Hilt (Dagger) for managing object lifecycles.
  • Persistence:
    • Room Database: For structured data (Apps, Notifications, Folders). Currently at Schema v14.
    • DataStore: For lightweight user preferences (Themes, Modes).
  • Asynchronous Logic: Kotlin Coroutines and Flow.

3. 📁 Module Breakdown (v4.1 Beta)

📱 UI Layer (com.echo.custom.ui)

  • MainActivity.kt: Refactored into a lean entry point. It delegates core responsibilities to specialized managers like WidgetBindingManager and PermissionLifecycleObserver.
  • screens/:
    • HomeScreen.kt: Now includes the Manual Voice Assistant Trigger button (positioned above pinned apps) and the v4.0 multi-state dashboard.
    • AppListScreen.kt: Enhanced with Search and Category filtering logic.
    • SaoResourceBlocks.kt: Upgraded with real-time thermal state and per-app memory telemetry components.

🧠 Logic Layer (com.echo.custom.viewmodel & managers)

  • NotificationIntelligenceManager.kt: The core intelligence orchestrator. It manages Google ML Kit's Entity Extraction and Language Identification pipelines.
  • HomeViewModel.kt: Now handles Voice Assistant command dispatching and the high-performance module filtering logic.
  • NotificationListenerHealer.kt: A self-healing manager that monitors the integrity of the notification listener service and performs re-binds if necessary.

💾 Data Layer (com.echo.custom.database)

  • EchoCustomDatabase.kt: Currently at Schema v15, including new columns for ML-derived sentiment and entity metadata in notification logs.
  • SystemRepository.kt: Expanded to track PowerManager.thermalState and granular app-specific memory consumption.

⚙️ Services (com.echo.custom.service)

  • VoiceAssistantService.kt: Manages the Speech-to-Text pipeline and the customizable Wake Word ("Echo") detection engine.
  • NotificationService.kt: Integrated with the Intelligence Manager for real-time triage and language-aware triage.

4. 🔑 Key Logic Deep Dives

🎙️ Voice Recognition Pipeline (v4.1 Beta)

The Maid Voice Assistant uses a multi-stage recognition flow to ensure privacy and efficiency:

  • Wake-Word Detection: A low-power listener monitoring for "Echo" (default) or user-defined triggers. This occurs 100% on-device.
  • Speech-to-Text: Once triggered, the VoiceAssistantService activates the Android SpeechRecognizer to process commands locally.
  • Intent Mapping: Recognized text is parsed and mapped to system actions (e.g., "Toggle HUD", "Enable Sleep Mode") via the HomeViewModel.

🧠 ML-Powered Intelligence (v4.0)

  • Entity Extraction: Uses Google ML Kit to scan notification content for actionable data (Dates, Addresses, Phone Numbers).
  • Language ID: Automatically identifies the language of incoming alerts to apply the correct sentiment analysis model (supporting EN, ES, FR).

🛡️ Service Healer & Integrity

  • Monitoring: The NotificationListenerHealer tracks the onServiceConnected state.
  • Restoration: If the service is unbound by the OS (e.g., due to memory pressure), the healer triggers a re-bind via a hidden ComponentName intent, ensuring notification features remain active.

⚔️ SAO Resource HUD

  • Thermal Telemetry: Maps PowerManager.thermalState to UI color shifts (Blue -> Red) and warning dialogues.
  • Memory Tracking: Fetches granular ActivityManager.MemoryInfo to display per-app consumption lists in a floating HUD.
