1. 🌟 Project Overview

EchoCustom Pro is a modular Android personalization suite and launcher. It combines several high-level features:

  • Sound Mapping: Custom audio overrides for notifications on a per-app/per-contact basis.
  • Smart Launcher: A feature-rich home screen with a "Maid" assistant, modular dashboard, and resource monitoring.
  • Elder Mode: A specialized, high-visibility UI for simplified accessibility.
  • Privacy & Security: Call screening and accessibility-based gestures.
Note: The Niagara Launcher/ directory in the root is a separate reference project (uIndex Launcher) and is not part of the Gradle build.

2. 🏗 Architecture & Tech Stack

The project follows modern Android development practices:

  • Language: 100% Kotlin (with some Java for legacy widget support).
  • Architecture: MVVM (Model-ViewModel-Intent) with a reactive data flow.
  • UI Framework: Jetpack Compose (Material 3) for all modern screens; XML for legacy widgets and TutorialActivity.
  • Dependency Injection: Hilt (Dagger) for managing object lifecycles.
  • Persistence:
    • Room Database: For structured data (Apps, Notifications, Folders). Currently at Schema v14.
    • DataStore: For lightweight user preferences (Themes, Modes).
  • Asynchronous Logic: Kotlin Coroutines and Flow.

3. 📁 Module Breakdown

📱 UI Layer (com.echo.custom.ui)

  • MainActivity.kt: The single-activity entry point. Configured with HOME category to act as a launcher. It manages navigation via NavHost, handles theme switching, and observes global states like Elder Mode.
  • screens/:
    • HomeScreen.kt: The main dashboard. Dynamically switches between the Professional Dashboard and the Elder Mode Grid.
    • AppListScreen.kt: A categorized list of all installed apps with search and filtering.
    • ContactScreens.kt: Manages contact-specific sound mappings and the dialer (in Elder Mode).
    • SaoResourceBlocks.kt: SAO-themed Compose UI components for monitoring CPU, RAM, and Network.
  • components/: Small, reusable UI units like MaidWidget, ResourceCard, and NotificationLogItem.

🧠 Logic Layer (com.echo.custom.viewmodel)

  • HomeViewModel.kt: The "brain" of the home screen.
    • Manages "Maid" dialogue logic based on system state (battery, time, RAM).
    • Integrates module filtering and state-aware visibility.
    • Handles app launching, folder management, and screen locking.
  • AppListViewModel.kt: Manages the retrieval and filtering of apps, including icon pack application and repair logic.

💾 Data Layer (com.echo.custom.database & com.echo.custom.data)

  • EchoCustomDatabase.kt: Defines the Room database with entities for AppEntity, NotificationLogEntity, FolderEntity, and WidgetEntity.
  • SystemRepository.kt: Core resource monitoring logic using /proc/stat and TrafficStats.
  • DataStoreManager.kt: Manages persistent preferences including the Maid Assistant toggle.

⚙️ Services & Receivers (com.echo.custom.service & com.echo.custom.receiver)

  • WidgetUpdateWorker.kt: Background state monitoring and voice playback (respects Maid toggle).
  • NotificationService.kt: A NotificationListenerService. Intercepts notifications, checks if a custom sound is mapped, and plays it.
  • EchoAccessibilityService.kt: Provides global gestures (like locking the screen on double-tap).

4. 🔑 Key Logic Deep Dives

🔊 Sound Mapping Logic

  1. A notification arrives -> NotificationService.onNotificationPosted is triggered.
  2. The service checks the AppDao for the package name.
  3. If customSoundUri exists and "Sleep Mode" isn't blocking it (or it's a VIP app), the sound is played.
  4. The notification is logged to the NotificationLog table for the dashboard.

⚔️ SAO Resource HUD

  • CPU Monitoring: Uses /proc/stat for real-time CPU wave generation.
  • Network Tracking: Uses TrafficStats for monitoring upload/download speeds.
  • Visual Fidelity: SAO-inspired design with high-contrast health bars for system resources.

🎭 Maid Assistant Toggle

  • Users can hide the Maid character and mute its background voice reactions via settings.
  • Preferences are persisted in DataStore and observed by WidgetUpdateWorker and HomeScreen.

👴 Elder Mode Transition

  • User toggles "Elder Mode" in settings.
  • DataStoreManager updates the elderModeEnabled Flow.
  • MainActivity observes this change and triggers a DashboardLoadingState.
  • HomeScreen recomposes, swapping the complex Professional Dashboard for a high-contrast app grid.

5. 🛠 Build & Environment

  • Target SDK: 35 (Android 15).
  • Java Version: JDK 17 required.
  • Gradle: 8.10.2 with AGP 8.7.2.
  • Widgets: Glance for modern interactive widgets and RemoteViews for legacy support.
