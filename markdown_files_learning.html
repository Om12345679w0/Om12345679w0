Project EchoCustom: Comprehensive Learning Guide

Welcome to the internal documentation for Project EchoCustom Pro. This guide provides a detailed breakdown of the project's architecture, logic, and file structure to help you understand how every piece fits together.

1. 🌟 Project Overview

EchoCustom Pro is a modular Android personalization suite and launcher. It combines several high-level features:

  • Sound Mapping: Custom audio overrides for notifications on a per-app/per-contact basis.
  • Smart Launcher: A feature-rich home screen with a "Maid" assistant, modular dashboard, and resource monitoring.
  • Elder Mode: A specialized, high-visibility UI for simplified accessibility.
  • Privacy & Security: Call screening and accessibility-based gestures.
Note: The Niagara Launcher/ directory in the root is a separate reference project (uIndex Launcher) and is not part of the Gradle build.

2. 🏗 Architecture & Tech Stack

The project follows modern Android development practices:

  • Language: 100% Kotlin (with some Java for legacy widget support).
  • Architecture: MVVM (Model-ViewModel-Intent) with a reactive data flow.
  • UI Framework: Jetpack Compose (Material 3) for all modern screens; XML for legacy widgets and TutorialActivity.
  • Dependency Injection: Hilt (Dagger) for managing object lifecycles.
  • Persistence:
    • Room Database: For structured data (Apps, Notifications, Folders). Currently at Schema v8.
    • DataStore: For lightweight user preferences (Themes, Modes).
  • Asynchronous Logic: Kotlin Coroutines and Flow.

3. 📁 Module Breakdown

📱 UI Layer (com.echo.custom.ui)

  • MainActivity.kt: The single-activity entry point. Configured with HOME category to act as a launcher. It manages navigation via NavHost, handles theme switching, and observes global states like Elder Mode.
  • screens/:
    • HomeScreen.kt: The main dashboard. Dynamically switches between the Professional Dashboard and the Elder Mode Grid.
    • AppListScreen.kt: A categorized list of all installed apps with search and filtering.
    • ContactScreens.kt: Manages contact-specific sound mappings and the dialer (in Elder Mode).

🧠 Logic Layer (com.echo.custom.viewmodel)

  • HomeViewModel.kt: The "brain" of the home screen.

💾 Data Layer (com.echo.custom.database & com.echo.custom.data)

  • EchoCustomDatabase.kt: Defines the Room database with entities.

⚙️ Services & Receivers (com.echo.custom.service & com.echo.custom.receiver)

  • NotificationService.kt: A NotificationListenerService. Intercepts notifications.

4. 🔑 Key Logic Deep Dives

🔊 Sound Mapping Logic

  1. A notification arrives -> NotificationService.onNotificationPosted is triggered.
  2. The service checks the AppDao for the package name.
  3. If customSoundUri exists and "Sleep Mode" isn't blocking it, the sound is played.
