Architecture Overview

LauncherBeta v3.0 follows a clean, reactive architecture built on the following pillars:

  • Unidirectional Data Flow (UDF) with ViewModels and Kotlin Flow.
  • Local-First Persistence using Room for layout state and DataStore for preferences.
  • Asynchronous Engine powered by Coroutines for all IO operations.

Resilience Logic

The core of v3.0 is its self-healing capability. This is achieved through:

Package Monitors

We use BroadcastReceiver and LauncherApps.Callback to listen for system-wide app changes. When an app is uninstalled, the CleanupWorker is triggered via WorkManager to scrub the database.

Safe Deserialization

Using Kotlinx.Serialization with custom Json configurations that ignore unknown keys and provide default fallbacks for corrupted fields.

Central Hub Search

The search engine in LauncherBeta is a hybrid system:

  • Local Index: Fast, prefix-based matching for installed applications.
  • Web Suggestions: Non-blocking network requests to DuckDuckGo API with 300ms debounce to minimize traffic.
  • Intent Dispatcher: Smart logic to determine if a query should open an app, trigger a web search, or calculate a mathematical expression.

Customization API

Developers can interact with LauncherBeta through:

  • IconPackManager: Standardized interface for loading resources from external APKs.
  • ThemeEngine: A Compose-based system that extracts color palettes from wallpapers using the Android Palette API.
